## 2025-02-28 - Optimizing Kubernetes I/O in Monitoring Loops
**Learning:** Found a specific performance bottleneck where the `_watch_airflow` loop internally refetched pod lists for every single pod status check, creating unnecessary `kubectl` subprocess overhead. Additionally, PVC cleanup in `_fix_statefulset_conflict` was executed sequentially. While Python logic loops are fast, repeated subprocess calls to a Kubernetes API proxy create substantial I/O drag during an ongoing monitoring loop.
**Action:** When monitoring a cluster via a CLI wrapper like `kubectl`, always fetch the full list of resources (`pods`, `jobs`) exactly once per loop iteration, and pass that cached state down into any internal helper methods (`_all_ready`, `_has_fatal_error`, etc.) to minimize network/subprocess overhead. Batch statefulset and PVC deletion using variable unpacks (e.g. `*statefulsets`) to execute them simultaneously rather than looping over individual deletes.
